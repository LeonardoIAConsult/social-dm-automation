# directives/ — Capa 1 (Qué hacer)

SOPs en Markdown. Cada archivo describe un procedimiento: objetivo, entradas,
herramientas de la Capa 3 a usar, salidas y casos extremos. Son documentos vivos:
se mejoran con cada aprendizaje, no se descartan.

El agente (Capa 2) lee estas directivas y delega en la ejecución (`src/`, Capa 3).

## Índice
- [`create_campaign.md`](create_campaign.md) — dar de alta/editar una campaña de DM.
- [`caption_driven_keywords.md`](caption_driven_keywords.md) — disparo por keyword derivada del copy del post.
- [`drive_delivery.md`](drive_delivery.md) — entrega de documentos desde Google Drive por fecha del post.

> No crear ni sobreescribir directivas sin pedir permiso, salvo indicación explícita.
